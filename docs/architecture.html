System Stages
6-Stage Processing Pipeline

From raw 20kHz vibration files to a live WebSocket inference stream — each stage has a clear interface and can be swapped independently.

01
⚙️ Signal Conditioning
Raw bearing vibration data ingested from CSV/binary. Bandpass filter (1–10kHz) removes DC offset and out-of-band noise. Z-score normalisation per channel. Automatic artifact rejection via IQR gating. Outputs clean 20kHz time-series windows.
signal_processing.py
↓
02
📐 Feature Engineering
14 hand-engineered features extracted per 0.1s window: RMS, peak, kurtosis, skewness, crest factor, spectral entropy, BPFO/BPFI harmonics (×2 each), sideband energy, envelope power. Sliding window stride: 32 samples.
features.py
↓
03
🔢 VibFormer Encoder
4-head multi-scale temporal attention over 30-step sequences. d_model=128. 3 feed-forward blocks with residuals and LayerNorm. CLS token aggregation for sequence-level representation. Learnable positional encoding.
vibformer.py
↓
04
⚡ Physics Regulariser
Monotonicity penalty enforces dH/dt ≤ 0 — bearing health must be non-increasing. Boundary condition loss anchors H(0)≈1 at start and H(t_fail)≈0 at failure. Combined loss: L = L_MSE + 0.1·L_mono + 0.05·L_boundary.
physics_loss.py
↓
05
📦 ONNX Export + INT8 Quantisation
PyTorch model exported to ONNX opset 17. Post-training INT8 static quantisation via ONNX Runtime. ~4× speedup over FP32 on CPU with <0.3% RMSE degradation. Model size reduced from 8.2MB → 2.1MB.
export.py
↓
06
🚀 FastAPI + React Dashboard
Async Uvicorn server exposes REST + WebSocket API. React dashboard consumes live telemetry via WS, renders RUL countdown, health arc gauge, fault event log, and 14-channel sensor grid using Recharts + Lucide.
src/backend/app.py · frontend/
Feature Engineering
14 Engineered Signal Features

Each feature targets a specific failure mode. Combinations of time-domain and frequency-domain statistics give the model multi-scale insight into bearing health.

FeatureDomainTargetsDescription
rmsTimeOverall degradationRoot mean square — primary energy indicator, increases monotonically near failure
kurtosisTimeImpulse faults4th statistical moment, highly sensitive to early-stage spalling and impact events
crest_factorTimeBall/race faultsPeak-to-RMS ratio, drops after initial spike as damage spreads uniformly
skewnessTimeAsymmetric wear3rd moment, captures directional asymmetry in vibration distribution
bpfo_1, bpfo_2FrequencyOuter race faultBall Pass Frequency Outer race at 1× and 2× harmonics
bpfi_1, bpfi_2FrequencyInner race faultBall Pass Frequency Inner race at 1× and 2× harmonics
spectral_entropyFrequencyDistributed damageShannon entropy of PSD — decreases as energy concentrates at fault frequencies
sideband_energyFrequencyModulation faultsEnergy in ±Δf sidebands around BPFO/BPFI, indicates bearing wobble
envelope_powerEnvelopeEarly faultsPower of Hilbert-transformed envelope signal, detects amplitude modulation
peak_2_peakTimeGross instabilityMax − min in window; rises sharply during pre-failure
