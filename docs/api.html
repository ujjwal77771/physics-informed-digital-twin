GET /health
System health check. Returns server status and whether the ONNX model is loaded and ready for inference. Use this before sending prediction requests to verify the server is live.
Response 200
{
  "status": "healthy",
  "model_loaded": true,
  "onnx_available": true,
  "api_version": "1.0.0"
}
curl http://localhost:8000/health
import requests
r = requests.get("http://localhost:8000/health")
print(r.json())
POST /predict
Run RUL prediction on a 30-timestep sliding window of 14 sensor feature channels. Returns predicted remaining useful life, health index, status classification, and maintenance recommendation string.
FieldTypeDescription
sequencefloat[30][14]30 consecutive timesteps × 14 engineered features. Each row is one measurement window.
Response 200
{
  "predicted_rul": 83.4,
  "health_index": 0.667,
  "status": "WARNING",
  "recommendation": "Schedule inspection within 48h"
}
import requests, numpy as np

sequence = np.random.randn(30, 14).tolist()

r = requests.post(
  "http://localhost:8000/predict",
  json={"sequence": sequence}
)
print(r.json())

WS /ws/telemetry
Real-time WebSocket stream. Pushes one JSON message per second simulating a bearing run-to-failure sequence. Includes raw sensor readings, model predictions, health index, and status. The React dashboard connects to this endpoint live.
Stream Message (JSON)
{
  "step": 42,
  "sensors": [0.12, -0.08, 0.34, ...],  // 14 values
  "true_rul": 83.0,
  "predicted_rul": 80.7,
  "health_index": 0.646,
  "status": "NORMAL"              // NORMAL | WARNING | CRITICAL
}
import asyncio, websockets, json

async def monitor():
  async with websockets.connect(
    "ws://localhost:8000/ws/telemetry"
  ) as ws:
    async for msg in ws:
      data = json.loads(msg)
      print(f"RUL: {data['predicted_rul']:.1f} | H(t): {data['health_index']:.3f}")

asyncio.run(monitor())
